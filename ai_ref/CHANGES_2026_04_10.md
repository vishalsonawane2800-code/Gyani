# Changes Made - 2026-04-10

## Issue
User encountered error when creating IPO: **"Could not find the 'abbr' column of 'ipos' in the schema cache"**

Logo upload also failing due to form submission errors.

---

## Root Cause Analysis

1. **Schema Mismatch**: Old migration (`001_create_ipo_tables.sql`) defined `abbr` column, but new v2 schema doesn't have it
2. **IPO Form Bug**: Admin form was still trying to include `abbr` field in form submission
3. **Schema Cache Out of Sync**: Supabase PostgREST wasn't aware of the schema change

---

## How Abbreviation Works Now

**Before (WRONG):**
1. User types company name
2. Auto-generates abbr
3. Submits abbr to database ❌
4. Database stores abbr ❌

**After (CORRECT):**
1. User types company name
2. Form doesn't track abbr
3. Form submission only sends name, not abbr ✅
4. Logo preview generates abbr on-the-fly from name ✅
5. Database never stores abbr ✅
6. When querying IPO, abbr is calculated: `company_name.split(' ').map(w => w[0]).join('').slice(0, 2).toUpperCase()` ✅

This is cleaner and matches the v2 schema design.
