# Changes Made - 2026-04-10

## Issue
User encountered error when creating IPO: **"Could not find the 'abbr' column of 'ipos' in the schema cache"**

Logo upload also failing due to form submission errors.

---

## Root Cause Analysis

1. **Schema Mismatch**: Old migration (`001_create_ipo_tables.sql`) defined `abbr` column, but new v2 schema doesn't have it
2. **IPO Form Bug**: Admin form was still trying to include `abbr` field in form submission
3. **Schema Cache Out of Sync**: Supabase PostgREST wasn't aware of the schema change

---

# Changes Made - 2026-04-11

## Issue
IPOs not appearing in the "Select IPO" dropdown in Manage Reviews section. Also bulk data entry (GMP, financials, peer comparison) is only available in Edit mode, not Add mode.

---

## Root Cause Analysis

1. **Column Name Mismatch**: Several admin pages were using `name` in their Supabase queries but the database uses `company_name`
2. **Bulk Data Design**: The bulk data entry is intentionally only in Edit mode because the IPO must exist in the database before you can add related data (financials, peers, GMP history)

---

## Files Fixed

### 1. `app/admin/reviews/page.tsx`
- **getIPOs()**: Changed `.select('id, name, slug')` to `.select('id, company_name, slug')` and transformed the result to map `company_name` to `name`
- **getReviews()**: Changed nested select to use `company_name` and transformed results

### 2. `app/admin/gmp/page.tsx`
- **IPO query**: Changed `.select('id, name, slug, ...)` to `.select('id, company_name, slug, ...)` and transformed results
- **GMP History query**: Changed nested `ipos (name, slug)` to `ipos (company_name, slug)` and transformed results

### 3. `app/admin/ipos/[id]/page.tsx` (IPO Detail View)
- Added transformation to map `company_name` to `name` for the client component

### 4. `app/admin/ipos/[id]/edit/page.tsx` (Edit IPO)
- Updated formData to use `ipo.company_name || ipo.name` for compatibility

---

## Why Bulk Data Entry is Edit-Only

This is **by design**:
1. First, create the IPO with basic details in the "Add New IPO" form
2. After the IPO is saved to the database with an ID, go to "Edit IPO"  
3. In the Edit page, the Bulk Data Entry sidebar allows you to add:
   - Financial Data (FY23, FY24, etc.)
   - Peer Comparison companies
   - GMP History entries

The bulk data requires an `ipo_id` foreign key, so the IPO must exist first.

---

## Data Flow Summary

| Admin Page | Database Column | Frontend Prop |
|------------|-----------------|---------------|
| All pages | `company_name` | `name` |
| Reviews | `company_name` via join | `ipos.name` |
| GMP | `company_name` via join | `ipos.name` |
| IPO Detail | `company_name` | `name` |

All transformations happen server-side to maintain consistent client-side interface.
